## 当前进度

路线图 A–K 的代码与协议验收范围已落地，当前数据库为 `schema_version=6`，跨端字段语义为 `data_format_version=1`。这里的“完成”指对应阶段的实现已入库，不等于所有后续能力（加密、PDF、远端增量同步）都已提供。

主线后已落地：

- AI 密钥可在设置写入系统凭据（环境变量优先）
- Faro API 已作为默认真实识图提供商：支持连接测试、图片多模态请求，以及提供商/模型跨重启保存；Mock 仅保留为离线测试选项
- Windows 主界面现代化（侧栏分页 + 统一主题）；设置可跟随系统或固定浅色/深色，Qt 控件与公式阅读页即时同步换肤
- 任务式录题第一阶段：新增工作台与独立录题模块，手动/AI 上传、处理、确认和入库在同一页面栈内连续完成
- 录题阻断修复：上传后即时大图/缩略图预览；回收站图片可重新录入；永久删除会安全清理 AI/审核关联和无引用原图
- 公式阅读与题目详情：AI 确认、题目详情和今日复习统一离线渲染 LaTeX；题库可双击或点击按钮进入主内容区详情页
- AI 网络韧性与复习闭环：Faro 临时断连自动退避重试且失败项可原任务续跑；今日复习已迁入主内容区，评分后连续切题
- 详情操作闭环：支持前后题、加入今日复习、收藏、归档、删除、回收站恢复和原图缩放，并保留题库/复习返回来源
- 题库局部刷新：单题录入、编辑、复习与状态操作不再清空整张列表；必要的全量刷新会恢复筛选、选中项和滚动位置
- AI 一图多题：兼容旧单题 JSON 与新的 `problems` 数组，同图候选逐题确认、独立入库并共享内容寻址原图
- AI 候选区域与人工纠偏：确认页高亮当前题目范围，可拖拽重画、框内移动、通过四边四角控制柄微调、恢复整图、拆分或合并
- 专用录题暂存：新 AI 录题在确认前不创建 `Problem/inbox`；手动表单与原图自动保存，可在关闭程序后继续
- 可发布基线第一轮：正式题目写操作补齐 Operation 日志与远端 create 落地；ZIP/`.ebpack`/`.gmshare` 恢复增加路径、体积、校验与回滚保护
- Windows wheel 已内置默认配置与协议 schema，并新增 Python 3.11–3.13 的 Windows CI、静态检查和安装后资源烟测

**详细条目与强制登记约定见 [`docs/08_完成更新记录.md`](docs/08_完成更新记录.md)。** 每完成一次可交付更新，必须先追加该文档，再刷新本段摘要。

当前边界与后续：

- `.ebpack` v1 当前仅支持未加密包；`protocol/encryption-v1.md` 仍是设计占位。
- Word 是当前稳定导出路径，PDF 导出尚未纳入可用能力承诺。
- Android 可导入 Windows `.ebpack`，但尚未实现 GitLink/GitHub 云下载或增量同步。
- GitLink/GitHub 当前以完整 Release 快照为主；LocalFolder 才提供已接入的 `changes/` Operation 通道。
- 插件、本地模型等仍属于可选后续方向。

协议入口：[`protocol/README.md`](protocol/README.md)；分享包规范：`protocol/gmshare-format-v1.md`；schema 变更：`docs/07_schema_v4_变更说明.md`。

## 快速开始（Windows）

```powershell
cd apps/windows
pip install -e ".[dev]"
python -m yancuo_win
```

首次使用真实 AI：打开“设置”，粘贴 Faro Key 并保存，填写模型广场中支持图片输入的模型 ID，依次点击“测试 Faro 连接”和“保存并应用 AI 设置”。密钥只保存在 Windows 凭据管理器；也可在启动前设置临时环境变量 `FARO_API_KEY`。

测试：

```powershell
cd apps/windows
pytest ../../tests -q
```

源码 checkout 运行时数据目录默认：`apps/windows/.yancuo_data/`；wheel 安装后默认使用当前用户数据目录。两者均可用 `YANCUO_DATA_ROOT` 覆盖。

## 原则摘要

- API 密钥不得明文入仓；配置只用环境变量/凭据引用名
- 第一阶段只做本地稳定功能，不提前做复杂多端同步
- 修改架构或数据格式前先说明兼容性影响
