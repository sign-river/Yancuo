## 如何追加

每条记录按时间倒序写在「更新日志」最上方，至少包含：

| 字段      | 说明                                        |
| --------- | ------------------------------------------- |
| 日期      | `YYYY-MM-DD`                                |
| 标题      | 一句话概括                                  |
| 类型      | 阶段验收 / 增强 / 体验 / 修复 / 文档        |
| 范围      | 模块或端（Windows / Android / protocol 等） |
| 要点      | 3–8 条用户可感知或架构级变化                |
| 验收/测试 | 如 `pytest` 结果、手工路径                  |
| 关联      | schema/协议/提交（可选）                    |

## 更新日志

### 2026-07-22 · 可发布基线第一轮加固

- **类型：** 增强 / 修复
- **范围：** Windows 应用服务、增量同步、备份与分享包、LocalFolder、打包与 CI
- **要点：**
  - 创建、编辑、标签、状态、复习、批量编辑、图片导入、AI/工作区接受与 `.gmshare` 导入统一写入题目 Operation 日志；首次拉取可落地远端 `create`
  - 同步快照覆盖状态、来源、复习、布尔、日期和标签字段；Operation 结构、审核提案与远端字段执行显式白名单及类型归一，冲突中的非冲突字段和标签并集可先自动合并
  - ZIP 统一拒绝路径穿越、符号链接、重复条目、加密条目和异常体积/压缩比；普通备份与 `.ebpack` 在替换前后校验数据库并支持失败回滚
  - LocalFolder 写锁增加独占创建、TTL、自愈和 `finally` 释放；仓库、Release、附件、设备和 tombstone 路径组件拒绝目录穿越
  - wheel 内置默认 TOML 与协议 schema；源码和安装态分别使用可预期的数据目录；新增 Windows Python 3.11–3.13 CI、ruff、单测和安装后资源烟测
- **验收/测试：** `python -m compileall -q apps/windows/src/yancuo_win`；`python -m ruff check apps/windows/src tests/unit`；`python -m pytest tests/unit -q` → `71 passed`；本地 wheel 构建、隔离安装与包内资源烟测通过
- **未验收项：** 当前环境未配置 JDK/`JAVA_HOME`，因此本轮未宣称 Android Gradle 测试通过
- **关联：** `protocol/data-format-v1.md`、`protocol/ebpack-format-v1.md`、`.github/workflows/windows-python.yml`
