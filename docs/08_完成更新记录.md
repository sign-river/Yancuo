## 更新日志

### 2026-07-23 · 公式渲染与题目专用详情页

- **类型：** 修复 / 产品逻辑 / UI 增强
- **范围：** Windows AI 确认、题库详情、今日复习
- **要点：**
  - 新增统一数学内容组件，将 `\(...\)`、`\[...\]`、`$...$` 和 `$$...$$` 在本地转换为 MathML，由 Qt WebEngine 排版，不依赖在线 CDN
  - 非公式内容先进行 HTML 转义；无法解析的公式回退为可读源码，避免单个坏公式导致整题空白
  - AI 确认页右侧拆为“阅读预览 / 编辑字段”，默认先显示渲染结果，编辑后自动刷新预览
  - 题库列表双击或选中后点击“打开详情”进入主内容区专用阅读页，仅保留应用左侧导航；页面同时展示原图、题干、答案、解析、错因和标签
  - AI 入库完成后的“查看刚入库题目”直接打开新详情页；详情页可返回题库或继续编辑
  - 今日复习改用同一渲染组件，隐藏答案时不会把答案内容写入页面 HTML
- **兼容边界：** 保持 `schema_version=4` 和既有题目字段；新增纯 Python 依赖 `latex2mathml`。复习整体迁入主内容区仍属于后续阶段
- **验收/测试：** `python -m pytest tests/unit -q` → `84 passed`；Ruff、compileall、`git diff --check` 通过；Qt offscreen 详情页烟测确认页面路由为专用详情索引且输出包含 MathML，截图确认极限与分式完成排版
- **关联：** `apps/windows/src/yancuo_win/ui/math_content.py`、`apps/windows/src/yancuo_win/ui/problem_detail.py`、`docs/09_UI与操作流程重构规划.md`

### 2026-07-22 · Faro API 默认真实识图接入

- **类型：** 功能 / 配置 / 体验
- **范围：** Windows AI Provider、设置、录题启动校验
- **要点：**
  - 默认 AI 提供商由 Mock 改为 Faro OpenAI 兼容接口，Base URL 固定为 `https://faroapi.com/v1`；Mock 保留为显式离线测试选项
  - 真实图片请求使用 `/chat/completions`，发送提示词与 Base64 `image_url` 多模态消息，并解析结构化 JSON 候选题
  - 新增 `/models` 连接测试，验证网络、Bearer Key 和用户填写的模型 ID；401/404/429 等错误给出针对性中文提示且不回显完整密钥
  - 设置页改为“Faro API（真实识图）”，提供商与图片模型写入本地 `preferences.json` 并在启动时恢复；文件不包含 API Key
  - Key 读取顺序对齐教程：临时环境变量 `FARO_API_KEY` 优先，其次 Windows 凭据管理器
  - AI 录题在创建收件箱暂存题目前先检查配置；缺少 Key 时直接提示，不再留下空白或回收站记录
- **兼容边界：** 保持 `schema_version=4`；具体图片模型 ID 由用户从 Faro 模型广场选择，程序不静默替换模型
- **验收/测试：** `python -m pytest tests/unit -q` → `80 passed`；Ruff、compileall、`git diff --check` 通过；测试覆盖 `/v1/models`、Bearer 认证、Base64 图片请求、JSON 结果解析、偏好持久化及缺 Key 无暂存副作用
- **参考：** `Faro_API_小白使用教程.docx`（2026-07-19 版）

### 2026-07-22 · 图片录题与回收站阻断修复

- **类型：** 修复 / 体验
- **范围：** Windows AI 录题上传、图片去重、题目永久删除
- **要点：**
  - AI 上传区改为大图预览与缩略图卡片，选择后即可核对图片；列表仅显示文件名，完整路径保留在悬浮提示中
  - 选择阶段校验图片可读性，无效文件不会进入识别任务
  - 原图重复检测排除回收站题目，允许同一图片在旧记录丢弃后重新开始录题
  - 清空回收站按依赖顺序删除审核项、AI 任务项、来源、题目、版本和图片记录，避免 schema v4 外键冲突
  - 空 AI 任务与临时提示词随题目清理；仍被其他题目复用的内容寻址图片不会误删，无数据库引用的物理文件在提交后清理
  - 永久删除失败时向界面返回可读错误并保持事务整体回滚，不再将 SQLAlchemy 调用栈直接暴露为操作结果
- **兼容边界：** 保持 `schema_version=4`，未修改跨端数据格式
- **验收/测试：** `python -m pytest tests/unit -q` → `76 passed`；Ruff、compileall、`git diff --check` 通过；Qt offscreen 图片预览/移除烟测通过；当前本地数据库在线副本验证“同图重录 + 清空 3 条回收站记录 + 共享原图保留”，`PRAGMA foreign_key_check` 为 0
- **关联：** `apps/windows/src/yancuo_win/ui/intake_page.py`、`apps/windows/src/yancuo_win/application/services.py`

- **类型：** 产品逻辑 / UI 重构 / 增强

  - AI 录题在同一页面栈完成图片选择、用户提示词、后台进度、原图对照、结果修改和最终入库
