# schema v5 变更说明

## 背景

AI 一图多题已经可以生成多个独立审核候选，但 schema v4 没有保存“候选题位于原图哪个区域”。因此确认页只能重复显示整张图片，人工也无法对 AI 的拆题结果进行纠偏。

## 迁移

| 项目 | 变更 |
| --- | --- |
| `schema_version` | **4 → 5** |
| `review_items.region_json` | 新增非空文本字段，默认 `{}` |
| 正式题目字段 | 不变 |
| `data_format_version` | 保持 **1** |

`region_json` 保存归一化矩形：

```json
{"x": 0.05, "y": 0.1, "width": 0.9, "height": 0.35}
```

- 原点位于原图左上角；
- 四个值均限制在 `0..1`，宽高不能越过图片边界；
- `{}` 表示没有可靠区域，界面按整张原图处理；
- 坐标属于 AI 审核过程元数据，不进入正式题目的跨端字段语义。

## 兼容性

- Windows 启动时自动执行加法迁移，不改写已有审核内容；
- schema v4 产生的候选迁移后 `region_json={}`，仍可确认、跳过或入库；
- schema v5 `.ebpack` 需要支持 schema v5 的客户端打开，旧客户端会按既有规则拒绝更高版本；
- Android 当前只消费正式题目，不需要理解候选区域；其包版本兼容上限同步提升为 schema v5，Windows 专用审核字段不进入 Android UI。

## 操作语义

- 拆分：沿当前区域较长方向等分，复制候选字段并创建独立暂存题；原图对象路径保持共享；
- 合并：仅允许同一审核会话且共享原图哈希的候选，文本字段依次拼接，标签取并集，区域取外接矩形；
- 重画：在等比例原图预览中拖拽生成新矩形，松开后立即覆盖当前候选的 `region_json`；恢复整图写回 `{}`；
- 微调：四边四角控制柄修改矩形边界，框内拖动整体平移；所有结果仍归一化并限制在原图内；
- 被合并候选标记为 rejected，其暂存题移入回收站，审计记录保留。
